BÀI TẬP 1:

    git commit --amend => dùng để thay đổi commit message (bấm i để vào chế độ insert và sửa message, nhấn esc để thoát chế độ insert, :wq thoát khỏi chế màn hình )
    git commit --amend-m”<message>” =>  ghi đè message commit gần nhất, không cần phải thao tác như lệnh git commit --amend
    git restore --staged <file> => đưa file đang ở vùng Staging Area về vùng Working Directory (có thể lựa chọn file để đưa về)
    git reset HEAD~1 => đưa các file của commit gần nhất của vùng Repository về vùng Working Directory (không thể lựa file để đưa về)

BÀI TẬP 2: 
    a. Tạo 3 file: file1, file2, file3
    b. Chạy lệnh: git init
    git sẽ tạo ra 3 vùng là Working Directory, Staging Area, Repository, và các file1, file2, file3 sẽ nằm ở vùng Working Directory 
    c. Chạy lệnh: git commit -m”init project”
    => Chạy được lệnh nhưng sẽ không có phiên bản nào được tạo, bởi vì chưa chạy lệnh git add, để add các file từ vùng Working Directory sang Staging Area.
    d. Chạy lệnh: git add file1
    => file1 từ vùng Working Directory sẽ được add vào vùng Staging Area
    e. Chạy lệnh: git commit -m”add file”
    => file sẽ được chuyển từ vùng Staging Area sang vùng Repository và tạo ra được phiên bản.
    f. Chạy lệnh: git add .
    => các file còn lại ở vùng Working Directory sẽ được chuyển sang vùng Staging Area
    g. Chạy lệnh: git restore --staged file2
    => file 2 sẽ được chuyển về vùng Working Directory
    h. Chạy lệnh: git commit -m”add all files”
    => file3 sẽ được commit ở lệnh git này
    i. Chạy lệnh: git reset HEAD~1
    => commit ”add all files” ở đây sẽ bị xóa và file3 sẽ trở về vùng Working Directory

BÀI TẬP 3: 

    a. Chạy lệnh git init
    => git sẽ tạo ra 3 vùng là Working Directory, Staging Area, Repository
    b. Tạo file file1.txt, file2.txt, oops/file3.txt.
    => các file ở vùng Working Directory
    c. Thêm file .gitignore, thêm vào 2 dòng:
    i. file1.txt
    ii. oops/
    => chỉ còn file2, file3, và .gitignore ở vùng Working Directory
    d. Chạy lệnh: git add .
    => file2, file3, và .gitignore được chuyển qua vùng Staging Area
    e. Chạy lệnh: git commit -m"init project"
    => file ở vùng Staging Area được commit và chuyển lên vùng Repository
